### Init Container (train-models)
- Runs before backend starts
- Checks for existing models in `/app/artifacts/ensemble_metadata.json`
- Trains models only if not found (idempotent)
- Uses DataSet.csv from `/data` volume
- Logs to MLflow
- Exits successfully to unblock backend startup
- Only runs once per deployment (restart: "no")

### Volume Persistence
All critical data persisted across restarts:
- model-artifacts: Trained models, scaler, imputer
- sqlite-data: Audit logs, scores, actions
- redis-data: Cache (optional)
- neo4j-data: Transaction graph
- mlflow-data: Experiment tracking
- prometheus-data: Time-series metrics
- grafana-data: Dashboards, settings

### Networking
- Custom bridge network: muleshield-network (172.28.0.0/16)
- Internal DNS resolution by service name
- All services communicate internally
- External access via port mappings:
  - 3000: Frontend (Nginx)
  - 8000: Backend (FastAPI)
  - 6379: Redis
  - 7474/7687: Neo4j
  - 5000: MLflow
  - 9090: Prometheus
  - 3001: Grafana

### Health Checks
All services include health checks with:
- Configurable intervals (5-10s)
- Timeouts (3-5s)
- Retry counts (3-5)
- Start periods (10-30s)
- Automatic restart on failure (unless-stopped)

### Service Dependencies
Proper startup order with condition checks:
```
train-models depends_on:
  - mlflow (condition: service_healthy)

backend depends_on:
  - redis (condition: service_healthy)
  - neo4j (condition: service_healthy)
  - mlflow (condition: service_healthy)
  - train-models (condition: service_completed_successfully)

frontend depends_on:
  - backend (condition: service_healthy)

prometheus depends_on:
  - backend (condition: service_healthy)

grafana depends_on:
  - prometheus (condition: service_healthy)
```

## Quick Start

```bash
# 1. Copy dataset
cp /path/to/DataSet.csv ./data/DataSet.csv

# 2. Start services
docker compose up --build

# 3. Access services
# - Dashboard: http://localhost:3000
# - API Docs: http://localhost:8000/docs
# - MLflow: http://localhost:5000
# - Grafana: http://localhost:3001 (admin/admin)
# - Prometheus: http://localhost:9090
# - Neo4j: http://localhost:7474 (neo4j/muleshield123)
```

## Notes

1. **First Run**: The init container (train-models) will take 5-10 minutes to train the ensemble models. Subsequent runs skip training if models exist.

2. **Resource Requirements**: Minimum 8GB RAM, 10GB disk space

3. **Production Ready**: Follow security checklist in DOCKER_SETUP.md before production deployment

4. **Volume Cleanup**: Use `docker compose down -v` to remove all data (CAUTION)

5. **Development Mode**: See DOCKER_SETUP.md for hot-reload configuration
